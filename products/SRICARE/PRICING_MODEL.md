## 1. Pricing Philosophy

### 1.1 Pricing Principles

| Principle              | Description                                    |
| ---------------------- | ---------------------------------------------- |
| **Accessible**         | Harga terjangkau untuk keluarga menengah Batam |
| **Transparent**        | No hidden fees, semua biaya jelas di awal      |
| **Competitive**        | 15-20% lebih murah dari kompetitor Jakarta     |
| **Fair to Caregivers** | Caregiver mendapat porsi wajar (85%)           |
| **Sustainable**        | Margin cukup untuk operasional platform        |

### 4.2 Payment Flow

```
┌─────────────────────────────────────────────────────────┐
│                    PAYMENT FLOW                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. User selects service & books                        │
│         ↓                                               │
│  2. App shows total price breakdown                     │
│         ↓                                               │
│  3. User selects payment method                         │
│         ↓                                               │
│  4. Redirect to payment (QRIS/e-wallet/VA)              │
│         ↓                                               │
│  5. User completes payment                              │
│         ↓                                               │
│  6. Midtrans sends webhook to SRICARE                   │
│         ↓                                               │
│  7. SRICARE confirms booking                            │
│         ↓                                               │
│  8. Caregiver receives job notification                 │
│         ↓                                               │
│  9. After service completion:                           │
│     - Funds held in escrow                              │
│     - User confirms completion                          │
│     - Platform releases payment to caregiver            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.3 Escrow System

**Purpose:** Protect both user and caregiver

| Stage                  | Funds Location     | Action                      |
| ---------------------- | ------------------ | --------------------------- |
| Booking                | User → Escrow      | Payment received            |
| Service Ongoing        | Escrow             | Held securely               |
| Service Complete       | Escrow → Caregiver | Released after confirmation |
| Dispute                | Escrow (frozen)    | Manual review               |
| Refund (if applicable) | Escrow → User      | Returned to user            |

**Escrow Terms:**

- Funds released to caregiver within **24 hours** after service completion
- If user doesn't confirm within **48 hours**, auto-release
- Dispute window: **72 hours** after service
- Refund policy: Full refund if cancelled **>24 hours** before, 50% if **<24 hours**
