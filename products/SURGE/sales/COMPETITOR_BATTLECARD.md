# BATTLECARD BY COMPETITOR

## vs ThingsBoard

### When Customer Mentions ThingsBoard

**Customer Says:** "ThingsBoard gratis dan open source"

**Your Response:**
> "ThingsBoard Community memang gratis, tapi untuk production perlu:
> - Server hosting: $50-200/bulan
> - Database: $30-100/bulan
> - DevOps maintenance: $500+/bulan
> - **Total: $600-1,400/bulan** vs SURGE $99 all-inclusive"

### ThingsBoard Weaknesses to Highlight

| Weakness | SURGE Advantage |
|----------|-----------------|
| Complex setup (weeks) | Setup dalam days |
| Steep learning curve | Intuitive UI |
| No local support | WA + Phone Indonesia |
| No KLHK compliance | Built-in KLHK format |
| Generic platform | Industry-specific modules |

### When ThingsBoard is Actually Better

- Customer has dedicated DevOps team
- Need extremely custom rule engine
- Already invested in ThingsBoard infrastructure

**Pivot:** "Untuk use case tersebut, ThingsBoard memang option. Tapi apakah worth the TCO?"

---

## vs Grafana

### When Customer Mentions Grafana

**Customer Says:** "Grafana visualisasinya lebih bagus"

**Your Response:**
> "Grafana memang excellent untuk visualization. Tapi Grafana adalah dashboarding tool, bukan IoT platform.
>
> Untuk IoT, Anda masih butuh:
> - InfluxDB/Prometheus untuk data storage
> - MQTT broker terpisah
> - Device management terpisah
>
> SURGE = semua sudah terintegrasi. Grafana bisa di-integrate via API jika butuh advanced viz."

### Grafana Weaknesses to Highlight

| Weakness | SURGE Advantage |
|----------|-----------------|
| Not IoT-native | Purpose-built for IoT |
| No device management | Full device CRUD |
| Usage-based pricing | Predictable monthly |
| No MQTT built-in | MQTT native |
| Requires technical expertise | Business user friendly |

### When Grafana is Actually Better

- Customer already has data pipeline (InfluxDB, etc.)
- Need extremely complex dashboards
- DevOps/data engineering team available

**Pivot:** "SURGE bisa feed data ke Grafana via API untuk best of both worlds."

- Local support untuk troubleshoot

**Lead with:**
- Multi-zone monitoring
- Cost allocation per tenant

**Key differentiator:** "Dashboard yang bisa diakses owner/tenant, bukan cuma operator"
